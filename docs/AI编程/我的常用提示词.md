## UI稿还原(next.js技术栈)
按视觉稿像素级还原Home页面的视觉以及交互，注意合理拆分模块。还原后，你需要截图对比还原的效果
- 1.这是视觉稿产物：C:\Users\Huya\Downloads\web game new\index.html。视觉稿的技术栈是html+css。你需要按照页面，合理拆分模块，按照我们项目的技术栈next.js + tailwind css，像素级还原。css一比一映射成对应的tailwindcss类，不得额外新增通用的tailwind css类。
- 2.这是用视觉稿起的本地服务：http://172.21.16.9:8080/，你可以基于这个服务截图
- 3.这是本项目起的本地服务：http://172.21.16.9:4000/
- 4.不要随意mock数据，特别是图片相关的。mock的数据统一放到src/mock目录管理。 如果是图片，可以统一用固定的链接：https://static-gameforge.huya.cn/dev/games/19530/preview/v0.1/1772173269540_01KJEW46H41C0B9081073FSJJE.thumbnail_new_small.webp?imageView2/1/w/360/h/560/format/webp/q/75/ignore-error/1
- 5.不要新增额外的交互，比如不得新增coming soon等
- 6.优先使用服务器组件，最小可交互元素才需要用客户端组件
- 7.图标优先使用src/assets/目录下的，如果没有可以复用的，则从figma导出svg的放到src/assets/目录下统一管理。同时导出的svg图标需要将fill等属性改成currentColor，去掉svg外层的width，和height。统一在使用的时候设置颜色和尺寸，比如： `<PlaysIcon aria-hidden="true" className="size-4 shrink-0 text-red-400" focusable="false" />`
- 8. tailwind css优先使用tailwind css提供的原子类，避免使用字面量，比如不要使用leading-[14px]，推荐改成leading-3.5
